# Tecnologias Necessárias para o Frontend do App

Este documento especifica as tecnologias recomendadas para o desenvolvimento do frontend do aplicativo, que é um dashboard simples com autenticação via Google, suporte para desktop e mobile (inicialmente como Progressive Web App - PWA), integração com Firebase (para auth, dados e hosting) e configurações para webhooks do TikTok e notificações via Telegram. O foco é em simplicidade, performance, responsividade e desenvolvimento rápido.

O frontend será construído como uma Single Page Application (SPA) para uma experiência fluida, com suporte a PWA para instalação em mobile e acesso offline parcial.

## Framework Principal
- **React**: 
  - Versão: 18 ou superior.
  - Motivo: Framework JavaScript leve e reativo para construção de interfaces componentizadas. Facilita gerenciamento de estado e integração com Firebase.
  - Uso: Componentes para dashboard, formulários e exibição de dados em tempo real.
  - Ferramenta de Inicialização: Vite (para builds rápidos e hot reloading) ou Create React App (CRA).

## Integração com Firebase
- **Firebase SDK**:
  - Pacotes: `firebase` (inclui auth, firestore, etc.).
  - Motivo: Para autenticação (Google Sign-in), queries real-time no Firestore e hosting.
  - Uso: Hooks como `useAuthState` para gerenciar login e listeners para updates de vendas.

## Gerenciamento de Estado e Dados
- **React Hooks Nativos**:
  - Inclui: `useState`, `useEffect`, `useContext`.
  - Motivo: Suficiente para um app simples; evita overhead de bibliotecas externas.
- **React Query** (opcional, mas recomendado):
  - Versão: Última estável.
  - Motivo: Para caching de dados do Firestore, handling de loading/errors e otimização de performance (reduz requisições desnecessárias).

## Roteamento e Navegação
- **React Router**:
  - Pacote: `react-router-dom`.
  - Motivo: Para navegação entre telas (ex.: login, dashboard, configurações).
  - Uso: Rotas protegidas com auth guards.

## Estilização e UI Components
- **Alternativa: Material-UI (MUI)**:
  - Pacote: `@mui/material`.
  - Motivo: Se preferir componentes prontos (botões, forms, tables) com theming responsivo.
  - Uso: Para um design mais polido sem CSS custom.

## Formulários e Validação
- **React Hook Form**:
  - Motivo: Gerenciamento eficiente de forms com validações integradas e performance otimizada (menos re-renders).
  - Uso: Para inputs de código de acesso, configs TikTok e Telegram (token e chat ID).

## Notificações In-App e Utilitários
- **React Toastify**:
  - Motivo: Para feedback visual in-app (ex.: "Configurações salvas com sucesso").
  - Uso: Complementa notificações externas via Telegram.

## Suporte a PWA (para Mobile e Desktop)
- **Vite PWA Plugin** (ou CRA PWA template):
  - Pacote: `vite-plugin-pwa`.
  - Motivo: Transforma o app em PWA para instalação em mobile, suporte offline e notificações web (se necessário no futuro).
  - Uso: Gera manifest.json e service workers para caching de assets.

## Tipagem e Qualidade de Código
- **TypeScript** (recomendado):
  - Motivo: Tipagem estática para evitar erros em runtime, especialmente com Firebase SDK.
  - Uso: Defina interfaces para props, estados e dados do Firestore.
- **ESLint + Prettier**:
  - Motivo: Manter código limpo, consistente e sem erros.
  - Uso: Configuração padrão para React + TypeScript.

## Requisições HTTP (se necessário)
- **Fetch API Nativo** ou Axios:
  - Pacote: `axios` (opcional).
  - Motivo: Para chamadas extras (ex.: testar webhooks), mas Firebase SDK cobre a maioria.

## Dependências de Desenvolvimento
- **Node.js**: Versão 18+ (para runtime).
- **npm ou Yarn**: Para gerenciamento de pacotes.
- **Firebase CLI**: Para deploy no Hosting e emuladores locais.

## Instalação e Setup Inicial
Para iniciar:
1. Crie o projeto: `npx create-vite@latest my-app --template react-ts`.
2. Instale dependências principais: `npm install react-router-dom firebase react-hook-form tailwindcss postcss autoprefixer react-query react-toastify`.
3. Configure Tailwind: Adicione ao `tailwind.config.js`.
4. Integre Firebase: Crie app no console e adicione config no código.

## Considerações Finais
- **Performance**: Use code splitting (React.lazy) para bundles menores. Teste responsividade em dispositivos reais.
- **Segurança**: Armazene tokens sensíveis (ex.: Telegram) apenas no backend/Firestore, não no frontend.
- **Escalabilidade**: Para crescimento, adicione testes com Jest e React Testing Library.
- **Versões**: Use as mais recentes estáveis para compatibilidade com Firebase (verifique changelogs em 2025).

Este setup permite um MVP funcional em poucos dias, com custos mínimos no Firebase Hosting. Ajustes podem ser feitos com base em requisitos específicos.
